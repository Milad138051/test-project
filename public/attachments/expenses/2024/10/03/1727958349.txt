
در یک سازمان افرادی حضوردارند که بابت هزینه کردهای خود در آن سازمان پرداختی داشته اند و یا قصد پرداخت هزینه دارد.
ما قصد ایجاد محیطی برای ثبت درخواست هزینه کرد داریم به نحوی که پس از ثبت ، درخواست جهت تایید/رد به کارتابل دیگری ارجاع داده شود و پس از تایید پرداخت هزینه به درخواست دهنده صورت گیرد. به این منظور باید قابلیت هایی در سیستم ایجاد گردد تا موارد ذیل در قالب آن انجام پذیرد.
فرضیات:
1-	لاگین و احراز هویت وجود ندارد.
2-	برای عملیات پرداخت API مربوط به 3 بانک موجود است.

مراحل:
1.	ثبت درخواست
لازم است فرمی جهت ورود اطلاعات در اختیار کاربر قرار گیرد. این فرم شامل موارد زیر است: 
1-1	انتخاب نوع هزینه کرد (دسته بندی، مانند : حمل و نقل، ایاب ذهاب، خرید تجهیزات و ...)
2-1	شرح 
3-1	مبلغ 
4-1	کاربر بتواند برای درخواست خود فایل پیوست اضافه کند.
5-1	شماره شبا 
6-1	کد ملی 
لازم است تا اطلاعات افراد در دیتابیس در جدولی جدا وجود داشته باشد و بر اساس کد ملی وجود فرد در دیتابیس بررسی شود. این جدول با اطلاعات و داده فیک پر شود.
همچنین دسته بندی هزینه کردها نیز موجود باشد و توسط فیلد انتخابی توسط کاربر انتخاب گردد.
2.	Api ثبت درخواست: 
در کنار فرم درخواست، یک Api نیز جهت ثبت درخواست در نظر گرفته شود. الزامی به استفاده از این Api در فرم وجود ندارد.
3.	تایید/رد درخواست
3-1	درخواست های ثبت شده در کارتابل نفر تایید کننده به همراه اطلاعات  مشاهده شود.
•	همچنین لازم است امکان دانلود فایل نیز در لیست فراهم باشد.
3-2	 امکان تایید و رد درخواست برای کاربر وجود داشته باشد.
•	در صورت رد درخواست امکان اضافه کردن توضیحات برای علت رد آن وجود داشته باشد.
•	امکان تایید و رد چند درخواست بصورت همزمان وجود داشته باشد.
4.	ارسال Notification برای کاربر درخواست دهنده
4-1	در صورتیکه درخواست رد شده باشد، از طریق پیامک و یا ایمیل به کاربر درخواست دهنده اطلاع داده شود.
نکته: نیازی به پیاده سازی وب سرویس برای ارسال پیامک نیست. صرفا کافی است در مکانی که Api ارسال پیامک می بایست اضافه شود، یک کامنت درج گردد. و همچنین برای ارسال ایمیل نیز نیاز به ارسال ایمیل واقعی نیست و صرفا کدهای ارسال در خروجی باشد و جهت جلوگیری از خطا، ارسال ایمیل کامنت شده باشد.
5.	پرداخت
5-1	 پس از تایید درخواست، لازم است امکان پرداخت هزینه به کاربر درخواست کننده از دو طریق صورت پذیرد:
1.	بصورت اتوماتیک و زمانبندی شده
هر روز یک بار رأس یک ساعت خاص موارد تایید شده، بصورت اتوماتیک برای پرداخت ارسال شود و پس از پرداخت موفق از طریق ارسال پیامک و یا ایمیل، به کاربر درخواست دهنده اعلام گردد.

2.	بصورت دستی 
کاربر تایید کننده بوسیله ی یک دکمه موارد تایید شده را برای پرداخت ارسال نماید. (همان عملکرد اتومات ولی توسط کاربر)
عملیات پرداخت: عملیات پرداخت به این صورت است که با توجه به شماره شبا، بانک مورد نظر انتخاب شود و Api مربوط به آن بانک فراخوانی گردد.
شناسایی بانک بر اساس کد شبا نیازی به تبعیت از الگوی واقعی نیست. صرفا کافی است بر اساس 2 رقم اول شماره شبای وارد شده، بانک مشخص شود. (بعنوان مثال اگر شماره شبا با دو رقم 11 شروع می شود مربوط به بانک شماره 1 است.)
Api بانک به اینصورت عمل می کند که از حساب شرکت برداشته و به حساب کاربر پرداخت می نماید (این Api یک Api فرضی است) 
لازم است در زمان ثبت هزینه این مورد چک شود که شماره شبای وارد شده متعلق به یکی از بانک های تعریف شده در سیستم باشد.
جهت سهولت کار می توانید مکانی را که نهایتا Api بانک فراخوانی می گردد، به شکل یک کامنت در نظر گرفته شود.
5-2	بصورت پیش فرض API مربوط به 3 بانک قابل استفاده است اما کد باید به نحوی باشد که امکان توسعه آن و  افزودن API به آن وجود داشته باشد
5-3	برای عملیات پرداخت، خطاهای بانکی در نظر گرفته شود.
5-4	امکان گذاشتن لاگ برای ردیابی خطاهای احتمالی حائز اهمیت خواهد بود.
5-5	لازم به ذکر است UI و UX حائز اهمیت نیست و در درجه پایین قرار دارند.
